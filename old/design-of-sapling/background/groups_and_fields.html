Groups and Fields

Groups and Fields

Much of the cryptography underpinning Zcash depends on algebraic structures with certain properties. Groups and Fields are two of these structures. There is a wealth of resources for learning about these structures on the Internet, but this document will provide an overview of the details which are relevant for Zcash.

Groups

We refer to a group G\mathbb{G}G as a set of elements S\textbf{S}S together with a binary operation +++ defined over those elements. We usually write group elements A∈GA \in \mathbb{G}A∈G with an uppercase letter. In order to be considered a group, several properties (the "group axioms") must hold:

  • Closure: The operation +++ must always produce another element of the group.
  • Associativity: Given any A,B,C∈GA, B, C \in \mathbb{G}A,B,C∈G, it must be that (A+B)+C=A+(B+C)(A + B) + C = A + (B + C)(A+B)+C=A+(B+C).
  • There must be an additive identity element 000 for which 0+A=A+0=A0 + A = A + 0 = A0+A=A+0=A for all A∈GA \in \mathbb{G}A∈G.
  • Any element AAA must have an additive inverse BBB such that A+B=0A + B = 0A+B=0. We usually call B=−AB = -AB=−A the "negative" of the element.

We often deal with "abelian" groups for which the following property also holds:

  • Commutativity: Given any A,B∈GA, B \in \mathbb{G}A,B∈G, it must be that A+B=B+AA + B = B + AA+B=B+A. In other words, the order of the operations doesn't generally matter, which is stronger than what is implied by the associativity property.

The simplest example of a group is the integers Z\mathbb{Z}Z under addition, which has an infinite number of elements. In cryptographic contexts, we typically work with groups with a finite number of elements. We refer to the number of elements in a group as its order.

Consider the finite group Zm\mathbb{Z}_mZ​m​​, which is defined as the integers Z\mathbb{Z}Z modulo some integer m>0m > 0m>0 under addition. In this group, the set S\textbf{S}S is the set of integers {0,1,2,...,m−1}\{0, 1, 2, ..., m - 1\}{0,1,2,...,m−1}, and A+BA + BA+B is the "remainder" of the division by mmm of the result. As an example, consider the group Z12\mathbb{Z}_{12}Z​12​​ where 121212 corresponds to the number of hours on a wall clock. In this group, 5+10=35 + 10 = 35+10=3; it is a "cyclic" group for which the addition operator "wraps around" the hours on a clock.

We cannot "multiply" group elements together, but we can "scale" them by adding them to themselves some number of times. As an example, we can scale a group element GGG by 333 by computing G+G+GG + G + GG+G+G. Generally, if an element G∈GG \in \mathbb{G}G∈G is scaled by a scalar sss, we write it as [s]G[s] G[s]G. We usually write scalars in lowercase letters.

If a group element G∈GG \in \mathbb{G}G∈G can be scaled to produce any other element in the group, we say that it is a generator of G\mathbb{G}G. If the group is of composite order, some elements will generate a smaller "subgroup" of the original group with order that is some factor of the original order. If the group is prime order, all nonzero elements generate the entire group. Group elements themselves have an "order" described as the number of elements they can generate through scalar multiplication.

Fields

We refer to a field F\mathbb{F}F as a set of elements S\textbf{S}S together with two binary operators +++ and ∗*∗ for which the following applies:

  • Closure: The operations +++ and ∗*∗ must always produce another element of the field.
  • Associativity: Given any a,b,c∈Ga, b, c \in \mathbb{G}a,b,c∈G, it must be that (a+b)+c=a+(b+c)(a + b) + c = a + (b + c)(a+b)+c=a+(b+c) and it must be that (a∗b)∗c=a∗(b∗c)(a * b) * c = a * (b * c)(a∗b)∗c=a∗(b∗c).
  • Commutativity: Given any a,b∈Ga, b \in \mathbb{G}a,b∈G, it must be that a+b=b+aa + b = b + aa+b=b+a and it must be that a∗b=b∗aa * b = b * aa∗b=b∗a.
  • Distributivity: Multiplication "distributes" over addition such that a∗(b+c)=(a∗b)+(a∗c)a * (b + c) = (a * b) + (a * c)a∗(b+c)=(a∗b)+(a∗c) for all a,b,c∈Fa, b, c \in \mathbb{F}a,b,c∈F.
  • There must be an additive identity element 000 for which 0+a=a+0=a0 + a = a + 0 = a0+a=a+0=a for all aaa.
  • There must be an multiplicative identity element 111 for which 1∗a=a∗1=a1 * a = a * 1 = a1∗a=a∗1=a for all aaa.
  • Any element aaa must have an additive inverse bbb such that a+b=0a + b = 0a+b=0. We usually call bbb the "negative" of the element.
  • Any element a≠0a \neq 0a≠0 must have a unique multiplicative inverse bbb such that a∗b=1a * b = 1a∗b=1. We sometimes call bbb the "reciprocal" of the element. Usually we just refer to this as the "inverse" if the context is clear.

The most common example of a field is the real numbers R\mathbb{R}R which contains an infinite number of elements. In fields, polynomials and other kinds of common arithmetic are all usable and have consistent properties. In cryptography we tend to work with fields with a finite number of elements for precision and space-efficiency reasons.

The easiest way to construct a finite field is to select some prime ppp and treat ZpZ_pZ​p​​ as a field under addition and multiplication modulo ppp. All of the above properties apply so long as ppp is prime. All finite fields have pkp^kp​k​​ elements for some prime ppp and some degree k>0∈Zk > 0 \in \mathbb{Z}k>0∈Z. We call ppp the characteristic of the field and typically write such fields as Fpk\mathbb{F}_{p^k}F​p​k​​​​.

Importantly, if we have an abelian group G\mathbb{G}G of prime order ppp, then the scalars for any G∈GG \in \mathbb{G}G∈G are elements of Fp\mathbb{F}_pF​p​​.

Field extensions

It is also possible to "extend" a field, much like how the complex numbers C\mathbb{C}C are an extension of the real numbers. All field extensions are themselves fields.

We describe C\mathbb{C}C formally as R[x]/(x2+1)\mathbb{R}[x] / (x^2 + 1)R[x]/(x​2​​+1). Since x2+1x^2 + 1x​2​​+1 is irreducible in R\mathbb{R}R (as −1\mathbb{\sqrt{-1}}√​−1​​​ does not exist in R\mathbb{R}R) the resulting elements a+bi∈Ca + bi \in \mathbb{C}a+bi∈C contain a "real part" aaa and an imaginary part bbb with i=−1i = \sqrt{-1}i=√​−1​​​. All field extensions are themselves fields.
